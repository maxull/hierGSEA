# hierGSEA v0.1.0-beta.1

First public beta release of `hierGSEA`.

`hierGSEA` is an R package for hierarchy-aware post-processing and
visualization of GSEA outputs generated upstream by `clusterProfiler`,
`ReactomePA`, or `clusterProfiler::GSEA()` with hierarchical custom databases.

## Highlights

- Reactome support with hierarchy-aware branch retention and ordering
- GO support for separate `BP`, `MF`, and `CC` ontologies
- MitoCarta support through the Broad `MitoPathways3.0` hierarchy
- local-family Benjamini-Hochberg correction aligned to visible hierarchy structure
- hierarchy-aware `ggplot2` dot plots with a left-side tree panel
- bundled single-fiber and HIRC example workflows
- pkgdown documentation scaffold with methods and output-object articles

## Recommended upstream settings

To preserve broad parent pathways in the source GSEA result, the recommended
starting settings are:

- `minGSSize = 5`
- `maxGSSize = 10000`
- `pvalueCutoff = 1`
- `pAdjustMethod = "none"`

## Beta status

This is a public beta release. The core workflow is usable, but plot defaults,
API edges, and some documentation details may still evolve as the package is
tested on more datasets.
